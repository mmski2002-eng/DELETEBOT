# 🔴 04 — Подделка NFT (NFT Spoofing) — CRITICAL

**Файл:** `nft-fixprice-sale-v4r1.fc:29-39, 130-166`  
**CWE:** CWE-345 (Insufficient Verification of Data Authenticity)

## Суть

Контракт хранит `nft_address` в статических данных (`static_data_cell`), но **НЕ проверяет**, что адрес NFT вычислен по стандарту TEP-62 как `hash(collection_address, item_index)`. Инициатор продажи может указать **любой** адрес в качестве NFT, в том числе фейковый контракт, который примет трансфер, но никогда не выпустит настоящий NFT из заявленной коллекции.

## Цепочка атаки

```
Атакующий:
  1. Создаёт фейковый NFT-контракт, который принимает transfer()
  2. Инициализирует sale с nft_address = фейковый_адрес
  3. Указывает в UI коллекцию X, item #42

Покупатель:
  4. Видит «NFT из коллекции X #42 за 100 TON»
  5. Отправляет 100 TON на контракт продажи
  6. transfer_nft(фейковый_адрес) → фейковый контракт принимает
  7. Покупатель получает фейковый NFT вместо настоящего из коллекции X
```

## Ущерб

Полная стоимость NFT — **не ограничена** (мошенничество с дорогими коллекциями).
