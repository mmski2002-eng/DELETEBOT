# 🟠 06 — Атака на истощение баланса через emergency_message (HIGH)

**Файл:** `nft-fixprice-sale-v4r1.fc:116-128, 402-427`  
**CWE:** CWE-400 (Uncontrolled Resource Consumption)

## Суть

Контракт не использует `raw_reserve()` перед серией выплат (`send_money` с флагом 3 = `PAY_FEES_SEPARATELY`). Комиссии блокчейна за каждое исходящее сообщение вычитаются из баланса контракта. При отправке 4 сообщений (3 `send_money` + 1 `transfer_nft`) комиссии могут исчерпать баланс до завершения цепочки, что приведёт к потере последних сообщений.

Для сравнения: `nft-offer.fc` содержит `raw_reserve(1000000, 0)` — правильную защиту.

## Цепочка атаки

```
1. Контракт продажи получает оплату (например, 1 TON)
2. Баланс контракта: 1 TON
3. send_money(owner, 0.9 TON)    → комиссия ~0.005 TON
4. send_money(fee_addr, 0.025)    → комиссия ~0.005 TON
5. send_money(royalty, 0.05)      → комиссия ~0.005 TON
6. transfer_nft(nft_addr, ...)    → комиссия ~0.005 TON + fwd_fee ~0.01 TON
                                     ↑ если баланс < комиссии → не отправлено
```

## Ущерб

Потеря NFT-трансфера при успешной оплате. Средства уже распределены, NFT не передан.
