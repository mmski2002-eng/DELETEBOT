## Пошаговый план атаки (Testnet)

### Фаза 1: Развёртывание двух версий (блок N)

```
1. Аудитор разворачивает уязвимый контракт продажи (БЕЗ raw_reserve):
   Контракт А:
   - nft_address = expensive-nft-item
   - full_price = 0.1 TON (маленькая цена для демонстрации)
   - marketplace_fee = 50000 (50%!)
   - royalty = 50000 (50%!)
   - ВАЖНО: user_amount = 0 (вся цена уходит на комиссии)

2. Аудитор разворачивает исправленный контракт продажи (С raw_reserve):
   Контракт Б:
   - Те же параметры
   - ДОБАВЛЕНА строка: raw_reserve(msg_value, 0) перед выплатами

3. Записываем начальные балансы всех участников
```

### Фаза 2: Атака на уязвимый контракт (блок N+1)

```
4. Покупатель отправляет на Контракт А:
   msg_value = 0.15 TON (цена 0.1 + 0.05 газа)
   op = buy

5. Контракт А выполняет buy_logic():
   fee_amount = 0.05 TON (50% от 0.1)
   royalty_amount = 0.05 TON (50% от 0.1)
   user_amount = 0 TON

6. Контракт А отправляет 4 сообщения:
   [A] send_money(Продавец, 0 TON)           → пропущено (amount=0)
   [B] send_money(Маркетплейс, 0.05 TON)     → комиссия ~0.005 TON
   [C] send_money(Автор, 0.05 TON)           → комиссия ~0.005 TON
   [D] transfer_nft(expensive-nft, Покупатель) → комиссия ~0.01 TON + fwd_fee ~0.02 TON
   
   Баланс контракта: 0.15 TON
   После [B]: 0.15 - 0.05 - 0.005 = 0.095 TON
   После [C]: 0.095 - 0.05 - 0.005 = 0.04 TON
   После [D]: Нужно 0.01 + 0.02 = 0.03 TON комиссии
              0.04 TON осталось → должно хватить?
   
   НО! Если комиссии больше (сложный маршрут):
   После [B] и [C]: 0.04 TON
   [D] требует: 0.01 (base) + 0.05 (fwd_fee для длинного маршрута) = 0.06 TON
   0.04 < 0.06 → СООБЩЕНИЕ НЕ ОТПРАВЛЕНО
```

### Фаза 3: Сравнение с исправленной версией (блок N+2)

```
7. Покупатель отправляет на Контракт Б (с raw_reserve):
   msg_value = 0.15 TON
   op = buy

8. Контракт Б выполняет buy_logic():
   raw_reserve(0.15, 0)  ← РЕЗЕРВИРУЕТ ВСЕ СРЕДСТВА
   fee_amount = 0.05 TON
   royalty_amount = 0.05 TON
   user_amount = 0 TON

9. Контракт Б пытается отправить:
   - send_money(Продавец, 0) — пропущено
   - send_money(Маркетплейс, 0.05) — НЕ МОЖЕТ (средства зарезервированы)
   → raw_reserve блокирует выплаты, если газа не хватит
   → КОНТРАКТ ПАДАЕТ С ОШИБКОЙ (throw)
   → ВСЯ ТРАНЗАКЦИЯ ОТКАТЫВАЕТСЯ
   → Покупатель сохраняет свои 0.15 TON
```

## Потенциально уязвимые контракты в Mainnet

### Признаки уязвимости

- Контракты продажи с `user_amount = 0` (вся цена уходит на комиссии)
- Контракты с высокой комиссией (> 25%)
- NFT, требующие дорогого газа на transfer (сложные NFT-контракты)
- Продажи с ценой < 1 TON (высокий риск нехватки газа)

### Поиск через API

```python
import requests

TONCENTER_API = "https://toncenter.com/api/v2"
SALE_CODE_HASH = "HASH_nft_fixprice_sale_v4r1"

resp = requests.get(f"{TONCENTER_API}/search", params={"code_hash": SALE_CODE_HASH})

for addr in resp.json()["result"]:
    sale = requests.get(f"{TONCENTER_API}/runGetMethod", params={
        "address": addr,
        "method": "get_sale_data"
    }).json()["result"]
    
    if not sale["is_complete"]:
        price = sale["full_price"]
        fee_pct = sale["fee_percent"]
        royalty_pct = sale["royalty_percent"]
        
        fee = (price * fee_pct) // 100000
        royalty = (price * royalty_pct) // 100000
        user = price - fee - royalty
        
        # Рискованные случаи
        risk_score = 0
        
        if user == 0:
            risk_score += 50  # Все деньги уходят на комиссии
        if price < 1_000_000_000:  # < 1 TON
            risk_score += 30  # Маленькая цена
        if fee_pct + royalty_pct > 50000:  # > 50%
            risk_score += 20  # Высокие комиссии
            
        if risk_score >= 50:
            print(f"РИСК: {addr} | Price: {price} | User: {user} | Fee%: {fee_pct} | Royalty%: {royalty_pct} | Score: {risk_score}")
```

```func

() recv_internal(...) {

    send_money(owner, amount);
```

## Ожидаемые результаты

### Если уязвимость подтверждена:
- Контракт без `raw_reserve` теряет сообщение transfer_nft при нехватке газа
- Деньги уже распределены, NFT не передан
- Контракт с `raw_reserve` корректно откатывает транзакцию

### Если опровергнута:
- Газа всегда хватает (маловероятно для граничных случаев)
- TON гарантирует достаточный газ для всех сообщений (опровергается практикой)
