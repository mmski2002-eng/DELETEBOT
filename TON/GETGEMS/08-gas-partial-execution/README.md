# 🟠 08 — Газовая атака через частичное выполнение (HIGH)

**Файл:** `nft-fixprice-sale-v4r1.fc:9, 172, 397-427`  
**CWE:** CWE-400 (Uncontrolled Resource Consumption) + CWE-252

## Суть

Контракт устанавливает `min_gas_amount = 0.1 TON` (строка 9), но:
- Не проверяет оставшийся газ между отправкой 4 сообщений
- Не использует `raw_reserve()` для защиты баланса
- Bounced-сообщения игнорируются (строка 172)

Атакующий может создать условия, при которых газа хватит на первые 3 сообщения (`send_money`), но не хватит на `transfer_nft`. Результат: деньги распределены, `is_complete=1`, NFT не передан.

## Цепочка атаки

```
Атакующий:
  1. Отправляет internal message с ценой + 0.1 TON
  2. Включает большой jetton_price_dict для увеличения потребления газа
  3. Выбирает сложный маршрут доставки сообщений

Контракт:
  4. buy_logic() — успех
  5. send_money(owner, ...)    → успех
  6. send_money(fee_addr, ...) → успех
  7. send_money(royalty, ...)  → успех
  8. transfer_nft(...)         → GAS EXHAUSTED (не хватило газа)
  9. save_data(is_complete=1)  → УЖЕ выполнено до transfer_nft
 10. Bounced приходит → строка 172: return (игнор)
```

## Ущерб

Полная стоимость NFT. Покупатель теряет средства, не получая NFT.
